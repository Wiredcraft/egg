Introduction

egg was built as an alternative to the many CSS/SASS frameworks out there; Foundation, Bootstrap and the likes. It intends to be minimalistic, solving as few problems as possible and providing developers with the basic defaults and mixins to solve the problems they experience designing their Web apps and websites.

It was designed with two core principles in mind;

  • Don't dictate HTML; while we do provide some defaults, we mostly provide mixins for users to decide how they want to associate style and markups rather than specific (and often heavy) classes and HTML structure.
  • Don't solve too many prolems; we focused on the few things that we think are difficult or important when building for the Web; typography, modals, forms... The rest is best solved by developerss for their specific use case.

Structure

At the root of the egg repo, you’ll find a few folders and files:

  • modules/ contains mostly mixins that you can use to build your various elements like tooltips or buttons. These are not very opinionated with regards to what HTML they are applied to. It also contains a file for all variables (modules/_varibles.scss) which
  • defaults/ contains leverages the mixins (from modules/) and provide sensible default ways of applying them to your HTML. This is definitely more opinionated and **is expected to be overriden by
  • vendors/ which contains (by default )
  • partials/

How to use it

We tried multiple ways of organizing and distributing egg (formerly an npm module called eggshell). We figured out that the best way for people to use a SASS boilerplate was to be able to override things, rather than use it as a dependency they would need to override to bend it to their will.

We recommend you leave the modules/ folder intact, however the rest is fair game. Change files in defaults/ or choose to not load them, override variables, build your own custom partials. Just go nuts.

Buttons

Simply use the .button class on elements (e.g. button, links) that you want to style as buttons:

<button class='button'>Default</button>
<button class='button' disabled>Disabled</button>
<button class='button primary'>Primary</button>
<button class='button secondary'>Secondary</button>

Additionally you can group buttons together with the .bundle class:

<span class='bundle'>
  <button class='button'>Option 1</button><button class='button'>Option 2</button><button class='button'>Option 3</button>
</span>

Notice that I put all elements on the same line, otherwise you will end up with spaces between your buttons (which you could fix with some float magic).

Customize

  • defaults/button.scss associates the .button, .button.primary, .button.secondary and .bundle classes.
  • modules/button.scss contains the button() and bundle() mixins.
