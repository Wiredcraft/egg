What is egg?

egg was built as an alternative to the many CSS/SASS frameworks out there; Foundation, Bootstrap and the likes. It intends to be minimalistic, solving as few problems as possible and providing developers with the basic defaults and mixins to solve the problems they experience designing their Web apps and websites.

It was designed with two core principles in mind;

  • Don't dictate HTML; while we do provide some defaults, we mostly provide mixins for users to decide how they want to associate style and markups rather than specific (and often heavy) classes and HTML structure.
  • Don't solve too many prolems; we focused on the few things that we think are difficult or important when building for the Web; typography, modals, forms... The rest is best solved by developers for their specific use case.

How do I use it?

  • main.css is a good starting point for adding new styles.
  • _variables.scss defines colors, font sizes, spacing & typography. Enough if you just want to make a few adjustments.
  • modules/ contains a few mixins that addresses some of the cor eissues (typography, spacing, buttons, etc).
  • vendors/ should contain 3rd party libraries (e.g. normalize.css).
  • partials/ is where you can organize code specific to your style (e.g. `partials/header.scss` for the header). It includes a few styles that are less often used with egg (e.g. the modal).

Simply copy the content of that folder wherever you want (for example the `_scss` folder in your app).

We recommend you leave the modules/ folder intact, however the rest is fair game: change files in partials/, tweak `_variables.scss` or choose to select includes and override styles in `main.scss`, that's up to you. Go nuts!
