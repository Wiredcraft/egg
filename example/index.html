What is egg? How do I use it? Examples

What is egg?

egg was built as an alternative to the many CSS/SASS frameworks out there; Foundation, Bootstrap and the likes. It intends to be minimalistic, solving as few problems as possible and providing developers with the basic defaults and mixins to solve the problems they experience designing their Web apps and websites.

It was designed with two core principles in mind;

  • Don't dictate HTML; while we do provide some defaults, we mostly provide mixins for users to decide how they want to associate style and markups rather than specific (and often heavy) classes and HTML structure.
  • Don't solve too many prolems; we focused on the few things that we think are difficult or important when building for the Web; typography, modals, forms... The rest is best solved by developerss for their specific use case.

How do I use it?

egg is organized as a single folder with a few key files and subfolders;

  • main.css is the main SASS file that pulls all of the mixins and styles together.
  • _variables.scss defines configurable variables; colors, font sizes, spacing, typography...
  • modules/ is mostly a series of mixins helping us solving the core issues; typography, spacing, buttons and forms interactions...
  • defaults/ is a collection of SASS files associating the mixins and styles from `modules/` with HTML markups and provide sensible defaults.
  • vendors/ contains styles for 3rd party libraries (by default it only contains normalize.css).
  • partials/ is where the code specific to your Web app or website resides. You would often find things like `_header.scss` to style the header of your app. By default, we only have `_twig.scss` which defines a font-face for the twig iconset.

Simply copy the content of that folder wherever you want (for example the `_scss` folder in your app).

We recommend you leave the modules/ folder intact, however the rest is fair game: change files in defaults/, tweak `_variables.scss` or choose to select includes and override styles in `main.scss`, that's up to you. Go nuts!
